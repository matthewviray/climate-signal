# dsc106-proj3

Design Rationale

Our visualization was designed around the idea that climate change is inherently both spatial and temporal. A single static plot can communicate overall trends, but it becomes difficult to compare how climate impacts vary across regions, scenarios, and decades simultaneously. We therefore designed an interactive plot that allows users to explore both geographic and temporal patterns together. 
The primary visual encoding uses choropleth-style gridded world maps to display projected climate anomalies. Color was chosen as the main encoding because anomaly magnitude is continuous and spatially distributed. Diverging color scales were used for temperature and precipitation because these metrics contain meaningful positive and negative deviations. We selected darker, higher-contrast colors to improve readability and make region selections visually distinct from the background climate field. 
The visualization uses coordinated multiple views. The line chart below the maps shows temporal trends for the currently selected scenario, while the maps show spatial distributions for selected decades. We initially considered using a single map with a draggable year slider, but we found that this made temporal comparison difficult because users had to mentally remember the appearance of previous years. The singular map resulted in one region being selected to be compared to the global mean, but we felt that the main goal of the plot should be to be able to visualize and quantitatively compare between regions, and not just the global mean that could be influenced by different outliers. To address this, we redesigned the interface so brushing a range on the line chart generates two synchronized maps side-by-side. This allows direct visual comparison between earlier and later decades without brushing back and forth between time ranges.
We also implemented selective interaction techniques. Users can select arbitrary geographic regions directly on the maps, and the line chart updates to show the selected region alongside the global mean. This interaction was chosen because it allows users to compare local climate trajectories against each other and also with worldwide averages. We considered predefined regional dropdowns, but free-form geographic brushing provided greater flexibility and encouraged exploratory analysis. 
Another major design decision involved simplifying scenario comparison. Earlier versions displayed average values for all scenario trajectories simultaneously, but this introduced visual clutter and reduced readability once multiple regional comparisons were added. The final design instead focuses on one selected scenario at a time while preserving multi-region comparison within that scenario. 
Finally, we included hover tooltips showing latitude, longitude, anomaly value, and decade to provide local detail without overwhelming the static view with labels. 

Development Process

	Our team was involved in multiple development processes, such as data processing, visualization implementation, and drafting/writeup. Raymond Wu and Matthew Viray handled exploratory data analysis, drafting of dynamic and interactive visualizations, while Taijun Park and Kyle Dang handled visualization modifications + rationale, and the development process write-up, respectively. Overall, we spent approximately 16.5 hours building the project, with data preprocessing and JSON export taking roughly 7 hours; map implementation taking 2 hours; line chart and coordinated view logic taking 1.5 hours; region selection and tooltip interactions taking 2 hours; styling and deployment taking 1 hour; and write-up taking approximately 3 hours. 
The most time-consuming aspect came from implementing the brushing interaction between the line chart and the dual maps simultaneously, as syncing the brush range on the time axis with the correct decade-averaged data on both maps required careful state management, with any update to the brush requiring continuous re-aggregation of the CMIP6 grid data and re-rendering both choropleth views without introducing lag or visual discrepancies. This was further complicated by the need to snap brush selections to the nearest valid decade boundary, which required rounding the selected year down to the nearest decade before updating the state and triggering a re-render.
Another challenge that came up during development was the geographic region selection on the map. Our decision to allow free-form spatial brushing over a gridded climate map, rather than predefined country outlines, required us to implement tracking mouse drag events over the SVG and filtering grid points by their latitude and longitude coordinates. This process involved inverting the Natural Earth projection at the brush endpoints to recover geographic coordinates, then filtering the CMIP6 data grid to only cells whose latitude and longitude fell within the selected bounding box. In doing so, the line chart can update responsively to reflect each (up to two) independent region selections across either map, while making the line chart’s responsive updates one of the more intricate parts of the overall frontend implementation.
	Before any of this could be rendered, all variables required preprocessing, with temperature and precipitation expressed as anomalies relative to a 1850-1900 baseline, precipitation being converted from kg per square meter per second to a percent anomaly, and data being aggregated into decadal averages and downsampled to a 48x72 grid, allowing us to keep file sizes manageable for the browser. An additional challenge arose from handling the sea ice concentration variable, which required a completely different rendering approach from temperature and precipitation. Since the siconc data is stored on CanESM5’s curvilinear ocean grid rather than a regular latitude and longitude grid, we implemented a KD-tree nearest-neighbor lookup to re-grid the data onto our 48x72 rectilinear grid before exporting. On the frontend, ice cells with concentration below 0.5% are filtered out entirely in order to avoid rendering noise over land and ocean mapping, requiring its own conditional data join separate from the temperature and precipitation rendering paths. Region brushing is also disabled for sea ice, since the curvilinear source grid makes geographic bounding-box selection unreliable.  
	The color palette was also a deliberate choice, as we used a diverging red-blue scale for temperature and a brown-blue scale for precipitation. Both variables, respectively, have a meaningful center at zero, where positive and negative shifts imply opposing real-world implications. For sea ice, we used a sequential blue scale since concentration meaningfully varies in one direction, and scenario colors for the three predictive trends (SSP1-2.6, SSP2-4.5, SSP5-8.5) were carried across both feature changes and the line chart to reduce cognitive load when switching between features.
